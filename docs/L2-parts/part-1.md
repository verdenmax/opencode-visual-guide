# L2 · Part 1 — 宏观全景 (The Big Picture)

**课程：** L01–L04 ｜ **状态：** 已完成

## 职责

Part 1 为整本指南建立**地图坐标系**：在深入任何子系统之前，先让读者看清 opencode 是什么、由什么构成、一次对话怎么流动、以及"两套引擎并存"这个贯穿全书的基本事实。它不深入实现细节，而是给出后面 60 课都能挂靠的骨架。

## 覆盖范围

| 课 | 主题 | 一句话 |
| --- | --- | --- |
| L01 | opencode 是什么 | 客户端/服务器架构的开源 AI 编程 agent；server 拥有一切，client 只是视图 |
| L02 | 项目全景地图 | ~24 个 package 分四组（CORE / 客户端 / 云端 / 基础库）+ 依赖方向 |
| L03 | 一次对话的生命周期 | keypress→render 的 11 跳：下行请求、中段 agent 循环、上行事件 |
| L04 | V1 与 V2 | 两套会话引擎并存；看路径定身份；V2 为主线 |

## 与相邻部分的关系

- **是 Part 2–12 的入口**：每个后续部分都在放大 L03 数据流中的某一跳。
- **L03 是全书的叙事主线**：agent 循环（Part 4）、System Context（Part 5）、LLM 协议层（Part 6）、工具（Part 7）、TUI（Part 10）都对应这趟旅程的具体一段。
- **L04 是读源码的钥匙**：V1（`packages/opencode/src/session`）vs V2（`packages/core`）的区分，决定了后面每一课该往哪个目录看。
- **L01 的"主线口诀"**（客户端发消息 → server 跑 agent 循环 → 事件流回客户端）在后续课程反复回指。

## 核心心智模型（读者读完应建立）

1. **server 拥有一切、client 是瘦视图**（L01）。
2. **CORE = core + llm + opencode(server/CLI) + server + sdk + plugin**，其余是周边（L02）。
3. **一次对话 = 下行收敛 + 中段循环（≤25 步）+ 上行事件**（L03）。
4. **看路径定 V1/V2，以 V2 为主线**（L04）。
